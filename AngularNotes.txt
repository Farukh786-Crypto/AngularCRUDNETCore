Here is a common use case where we change the background color of a directive when the mouse enters or leaves the host element:

TypeScript
import { Directive, HostListener, HostBinding } from '@angular/core';

@Directive({
  selector: '[appHighlight]'
})
export class HighlightDirective {

  @HostBinding('style.backgroundColor') backgroundColor: string = 'transparent';

  // Listen for the 'mouseenter' event on the host element
  @HostListener('mouseenter') onMouseEnter() {
    this.backgroundColor = 'yellow';
  }

  // Listen for the 'mouseleave' event on the host element
  @HostListener('mouseleave') onMouseLeave() {
    this.backgroundColor = 'transparent';
  }
}

ReactiveX
- A library used for async programming with observable streams

RxJs
  Observable
  - Representation of set of values
  - function that can return stream of values over the time (producer)
  - Values can range from 0 to infinity
  - Observable are called lazy, coz if there is no subsvcriber, it will not publish data
  - This are unicast, Each subscriber gets independent instance of observable
  Observer
  - This are the one that listen to your produces data & notify the subscribers
  - It has 3 callback function, it talks about different notification delivered by observable
    next() - success info
	error() - error info
	complete() - a flag indicating end of stream
  Operator
  - Function that takes 1 observable as input & return another observable after manipulation
  - pipe(op1, op2, op3)
  
  Subject
  - Special type of observable that allows mulitcast to multiple observers (subscribers)
  - It acts both producer & subscriber
  - IT can emit always new values to subscriber like event emitter
  
  BehaviorSubject
  - It is similar to regular subject but it can maintain the initial value.
  - It makes sure that whoever is subscribing to the BehaviourSubject will ateast get last data always available.
  
  ReplaySubject
  - It is similar to regular subject.You can configure to hold x values for a period of time.
  - It can maitain a buffer size. 
  - IT makes sure that whoever is subscribing to the ReplaySubject will alteast get those x no of data that is already stored.
  
1. Subject (The "Live Radio")
A standard Subject is multicast, but it has no memory. If you tune into a radio station late, you don't hear the songs that played before you joined; you only hear what is playing right now.

Theory: It does not store values. Subscribers only receive data emitted after they subscribe.

Best for: Event bus logic, like clicking a "Global Save" button.

TypeScript
const subject = new Subject<number>();

subject.next(1); // Lost (no one is listening)

subject.subscribe(val => console.log('Sub A:', val));

subject.next(2); // Sub A gets 2
2. BehaviorSubject (The "Last Played")
This is the most common Subject in Angular. It always has a "current value." When you subscribe, it immediately gives you the latest value it emitted.

Theory: It requires an initial value. It stores the latest value and emits it to new subscribers immediately upon subscription.

Best for: Storing "Current User" info or "Form State" in a service.

TypeScript
// Requires an initial value (0)
const bSubject = new BehaviorSubject<number>(0); 

bSubject.subscribe(val => console.log('Sub A:', val)); // Immediately gets 0

bSubject.next(10); // Sub A gets 10

bSubject.subscribe(val => console.log('Sub B:', val)); // Immediately gets 10
3. ReplaySubject (The "DVR / Catch-up TV")
This Subject acts like a recorder. It remembers a specific number of previous values and "replays" them for any new subscriber.

Theory: It does not need an initial value. You define a buffer size (how many items to remember).

Best for: Caching a list of data from an API so that multiple components can get the same history without re-calling the server.

TypeScript
// Buffer size of 2 (Remember last 2 values)
const rSubject = new ReplaySubject<number>(2);

rSubject.next(1);
rSubject.next(2);
rSubject.next(3);

// Subscriber joins late
rSubject.subscribe(val => console.log('Sub A:', val)); 
// Output: 2, 3 (It missed 1, but got the last 2)

	1. Array Methods Cheat Sheet (Data Manipulation)

	Method				Theory							Coding Example						Expected Result (The Output)

	map()			Creates a new array by 				[1, 2, 3].map(x => x * 2)						[2, 4, 6]
					transforming every element.
	

	filter()   		Returns all items that pass 		[10, 20, 30].filter(x => x > 15)				[20, 30]
					a specific condition.

	find()			Returns the first matching 			emps.find(e => e.id === 1)					{id: 1, name: 'John'}
					element (or undefined).

	findIndex()		Returns the index of the first		['a', 'b', 'c'].findIndex(x => x === 'b')		1
					 match (or -1).

	some()			Returns true if at least one		[1, 2, 3].some(x => x > 2)						true
					 item matches.


	every()		Returns true only if all items match.	[1, 2, 3].every(x => x > 0)						true


	reduce()	Condenses array into a single 			[1, 2, 3].reduce((acc, val) => acc + val, 0)	6
				value (like a sum).

	includes() 	Checks if a primitive value exists 		['Red', 'Blue'].includes('Red')					true
				in the array.

	slice()		Returns a copy of a portion 			['a', 'b', 'c'].slice(0, 2)						['a', 'b']
				(Original safe).

	splice()	Removes/Adds items (Modifies original).	arr.splice(0, 1) // Remove 1 item at index 0	[The removed item]




	2. String Methods Cheat Sheet (Text Formatting)


	split()				Converts a string into an Array.	"A,B,C".split(",")				["A", "B", "C"]

	trim()				Removes spaces from start and end.	" Hello ".trim()				"Hello"

	toLowerCase()      Makes all letters small 				"ADMIN".toLowerCase()			"admin"
						(good for comparison).

	includes()			Checks if a substring exists 		"Angular".includes("ng")		true
						inside a string.

	replace()			Replaces the first match with 		"Hi John".replace("John", "Doe")  "Hi Doe"
						new text.

find vs filter: find returns the object itself; filter returns an array containing the object.

some vs includes: includes only works for simple values (like 5 or "Hi"); some works for complex objects (like emp.id === 5).

map vs forEach: map returns a new array (Functional); forEach returns nothing (used for loops/logs).
